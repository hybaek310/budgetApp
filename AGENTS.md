# AGENTS.md

## 프로젝트 설명

이 저장소는 CSV 기반 가계부 CLI 앱 프로젝트다. 사용자는 CSV 파일을 읽고, 거래 내역을 조회/집계/검증하는 Python CLI 앱을 개발한다.

## 코딩 규칙

- 모든 함수와 메서드에는 타입 힌트를 반드시 사용한다.
- 함수 하나의 길이는 최대 50줄로 유지한다.
- 책임이 커지면 함수를 분리하고, 작은 단위로 테스트 가능하게 만든다.

## TDD 규칙

- 구현 전에 반드시 테스트를 먼저 작성한다.
- 실패하는 테스트를 확인한 뒤에 최소 구현을 작성한다.
- 리팩터링은 테스트가 모두 통과한 상태에서 진행한다.

## 품질 규칙

- 순환복잡도는 10 이하로 유지한다.
- 조건 분기와 중첩을 줄이고, 조합 가능한 작은 함수로 분해한다.

## 품질 검수 규칙

- 커밋 전에 반드시 `qa_engineer` 서브에이전트로 품질 검수를 진행한다.
- 서브에이전트 검수 결과에서 지적된 사항은 반영하거나, 반영 불가 사유를 명확히 기록한다.

## 테스트 실행 명령어

- `pytest`
- `radon cc`

## 커밋 규칙

- 하나의 기능이 개발되면 즉시 커밋하고 푸시한다.
- 커밋은 기능 단위로 작게 유지하고, 메시지에는 변경 의도를 분명히 적는다.
